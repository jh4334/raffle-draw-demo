# 인수 조건 (Acceptance Criteria)

## AC-1: 이름 파싱
- [x] 줄바꿈(`\n`)으로 분리한다.
- [x] 각 이름의 앞뒤 공백을 제거한다.
- [x] 빈 줄(공백만 있는 줄 포함)을 무시한다.
- [x] 중복 이름을 제거하고 첫 등장 순서를 유지한다.

## AC-2: 당첨자 수 검증
- [x] 당첨자 수가 1 미만이면 `RangeError`를 발생시킨다.
- [x] 당첨자 수가 참가자 수를 초과하면 `RangeError`를 발생시킨다.
- [x] 유효 범위(1 ≤ count ≤ names.length) 내이면 에러 없이 통과한다.

## AC-3: 무작위 비중복 추첨
- [x] 정확히 요청한 수만큼 당첨자를 반환한다.
- [x] 당첨자 간 중복이 없다.
- [x] 모든 당첨자는 원본 참가자 목록에 존재한다.
- [x] 전원 추첨(count === names.length) 시 전원이 반환된다.

## AC-4: 웹 UI
- [x] "뽑기" 클릭 시 결과가 `#result-list`에 `<li>`로 표시된다.
- [x] 유효하지 않은 입력 시 `#message`에 안내 문구가 표시된다.
- [x] "초기화" 클릭 시 입력·결과·메시지가 모두 초기 상태로 돌아간다.

## AC-5: 테스트
- [x] `npm test` (= `node --test tests/*.test.js`) 실행 시 전체 통과한다.
- [x] 테스트는 외부 의존성 없이 `node:test` + `node:assert`만 사용한다.
